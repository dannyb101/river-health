Calculation and Standards

When you press the Run Simulation on the application it will take the (log-normal) mean and s.d. of river flow that you have input and convert them to the mean and s.d. of the underlying normal distribution, using the algebra outlined here.

The application then uses that normal mean and s.d. to generate a log-normal distribution, and picks a river flow from that distribution.

The application also picks a random value of BOD and NH3 concentration, except from a normal distribution generated from their mean and s.d. concentrations.

These picks are performed for every row of CSO spill data. Each row of spill data therefore gets three values; a river flow, a BOD concentration and an NH3 concentration.

Once the river data has been added to the spill data the calculator performs a mass balance calculation on each row, using the formulas shown below.


The mass-balance calculations result in values for the downstream concentration of BOD and NH3 for each row. This produces a table of results which you can access by clicking Download Output Data on the outputs page. See below for an example of one of these tables. Note that rows where the CSO flow was zero, have been omitted. Also note that the calculator has summed each 5 minute interval of data into hourly intervals of data, where each hourly interval begins on the hour.


The calculator then uses this to calculate 99th percentile concentrations of each of the pollutants. This is the concentration of pollutant at which the river is either equal to or below, for 99 percent of the measurements made.

The 99th percentile is calculated for both the upstream and downstream river i.e. pre-spill and post-spill. This is shown in a box on the outputs page, pictured below.


As you can see, the river has been given a Water Framework Directive score both before and after the spill. This is based on tables 5.4(c) and 5.4(d) of the WFD framework found here and pictured below.


If any of the values change in the framework, you can navigate to the standards page and update them in the application.

Just below the WFD standards, the in-class deterioration is shown. This is used to calculate the Storm Overflow Assessment Framework (SOAF) impact score, based on table 7 of the SOAF guide, which is pictured below.


Finally, the impact classification is shown, which is based on table 11 of the SOAF guide, pictured below.


If you would like to improve the accuracy of the modelling of the river, you can increase the number of simulations. This means that the above process of finding the 99th percentile is repeated the number of times you specify in the number of simulations field and the results are averaged, as is commonly done in Monte Carlo simulations.

Note that while increasing the number of simulations does improve the accuracy of the model, results tend to converge at about 10 simulations.
